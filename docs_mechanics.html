📡 EventBus — system komunikacji

EventBus to centralny "magistrala zdarzeń" całej gry. Zamiast żeby skrypty trzymały bezpośrednie referencje do siebie nawzajem, każdy publikuje zdarzenia (Publish) albo nasłuchuje (Subscribe) — i nigdy nie wie kto po drugiej stronie.

Jak to działa

Skrypt A → EventBus.Publish(new OnPlayerDamaged { amount = 20f })
EventBus → szuka wszystkich nasłuchiwaczy OnPlayerDamaged i wywołuje ich callback
PlayerHUD → odbiera event i aktualizuje pasek HP

📊 Statystyki gracza

PlayerStats to ScriptableObject — jeden plik asset w projekcie, nie MonoBehaviour. Przechowuje wszystkie liczby gracza: bazowe wartości ustawiane w Inspectorze oraz runtime (bieżące HP, XP, poziom) i bonusy z ekwipunku.

PlayerStats to czyste dane — zero EventBus.Publish w środku. Zdarzenia publikuje wyłącznie PlayerHealthComponent.

Wartości bazowe (ustawiane w Inspectorze)

PoleDomyślnieOpis
baseMaxHp100Maksymalne HP na poziomie 1
hpPerLevel15Przyrost HP za każdy level
baseMaxStamina100Maksymalna stamina
staminaRegenPerSecond20Regeneracja staminy co sekundę

Wzory obliczeniowe

MaxHp = baseMaxHp + (hpPerLevel × (level − 1)) + bonusMaxHp
TotalDamage = baseDamage + (damagePerLevel × (level − 1)) + bonusDamage
TotalArmor = Clamp01(baseArmor + bonusArmor) // max 100%
TotalAttackSpeed = baseAttackSpeed + (attackSpeedPerLevel × (level − 1)) + bonusAttackSpeed
TotalCritChance = Clamp01(baseCritChance + bonusCritChance)

XP do następnego levelu = xpForLevel2 × (1.4 ^ (level − 1))

Bonusy z ekwipunku

Pola bonus* (bonusDamage, bonusArmor, bonusCritChance, bonusMoveSpeed, bonusMaxHp, bonusAttackSpeed) są resetowane i przeliczane przez InventorySystem.RecalculateBonuses() za każdym razem gdy gracz zakłada lub zdejmuje przedmiot.

Level up

Przy awansie HP gracza skaluje się proporcjonalnie (nie leczy, nie traci): currentHp = (currentHp / oldMaxHp) × newMaxHp. Gracz dostaje też 1 punkt umiejętności (availableSkillPoints++).

❤️ Zdrowie i stamina

PlayerHealthComponent to warstwa logiki i eventów na górze czystych danych z PlayerStats. Implementuje IDamageable i IHealable — wrogowie trafiają gracza przez te interfejsy bez twardej referencji.

Przepływ obrażeń

EnemyAI → playerDamageable.TakeDamage(10f)
PlayerHealthComponent → sprawdza IsInvincible → woła stats.TakeDamage() → redukuje przez Armor
EventBus → publikuje OnPlayerDamaged { amount, currentHp, maxHp }
PlayerHUD → aktualizuje pasek HP

Iframes (nieśmiertelność podczas uniku)

PlayerController ustawia playerHealth.IsInvincible = true na czas trwania uniku. TakeDamage() zwraca wtedy 0 bez żadnego efektu.

Regeneracja staminy

Co klatkę w Update() PlayerHealthComponent wywołuje stats.RegenerateStamina(deltaTime). Jeśli stamina wzrosła, publikuje OnPlayerStaminaChanged → HUD aktualizuje pasek staminy.

Wzór redukcji obrażeń

reduced = rawDamage × (1 − TotalArmor)
reduced = max(reduced, 1) // zawsze co najmniej 1 obrażenie

🏃 Ruch i unik

Gra jest click-to-move w stylu Diablo (izometryczny rzut). Ruch oparty na NavMeshAgent — nie physics-based. PlayerController obsługuje input i cel, steruje agentem.

Kliknięcie — priorytety

Co trafiłoEfekt
UI (EventSystem)Ignorowane — klik nie przechodzi do gry
Warstwa wrogów (EnemyLayer)Ustawia cel ataku, gracz podchodzi automatycznie
Warstwa ziemi (GroundLayer)Gracz idzie do punktu, cel ataku czyszczony

Podążanie za wrogiem

Gdy gracz ma ustawiony cel, co klatkę sprawdza odległość. Jeśli większa niż attackRange (1.5m) — agent idzie dalej. Jeśli mniejsza — agent zatrzymuje się, gracz obraca się w stronę wroga i PlayerCombat może atakować.

Unik

Wejście: Shift + Prawy przycisk myszy

Sekwencja uniku:

  1. Sprawdza czy jest wystarczająco staminy (dodgeCost = 25)
  2. Odejmuje staminę, włącza iframes (IsInvincible = true)
  3. Wyłącza NavMesh (updatePosition = false) — ruch przez Transform
  4. Gracz leci 0.3s z prędkością 12 w kierunku aktualnego ruchu (lub do przodu)
  5. NavMesh.SamplePosition — gracz wraca na siatkę NavMesh
  6. Iframes wyłączone, NavMesh wznowiony

Obrót gracza

Gracz obraca się przez Quaternion.Slerp z prędkością 15/s. Podczas uniku i gdy ma cel ataku — nie obraca się za ruchem agenta, tylko w stronę celu.

⚔️ Walka gracza

PlayerCombat siedzi na tym samym GameObject co PlayerController (RequireComponent). Nie ma własnego inputu — czyta stan z kontrolera przez dwa property: CurrentTarget i IsInAttackRange.

Cykl ataku

PlayerCombat.Update → sprawdza: canAttack && IsInAttackRange && CurrentTarget != null
TriggerAttack() → ustawia canAttack = false, ustawia AttackSpeed, odpala trigger "Attack"
Animation Event → OnAttackHit() — klatka uderzenia → nakłada obrażenia
Animation Event → OnAttackEnd() — koniec animacji → canAttack = true, reset cooldown

Obliczanie obrażeń

damage = TotalDamage // base + level + bonusy z ekwipunku
if (Random.value < TotalCritChance) damage *= critMultiplier // krit!
damage = Round(damage)
target.TakeDamage(damage, isCritical, knockbackDirection)

Fallback timer

Gdy animator nie ma skonfigurowanych Animation Events, fallback timer liczy 1 / TotalAttackSpeed sekund i po tym czasie resetuje canAttack. Gra działa bez animatora.

Animator — parametry walki

ParametrTypOpis
AttacktriggerOdpalany przy każdym ataku. Reset w OnAttackEnd i TickFallback.
AttackSpeedfloatMnożnik prędkości animacji = TotalAttackSpeed. Aktualizowany po equipie/levelu.
ResetTrigger: animator.ResetTrigger("Attack") jest wywoływany w OnAttackEnd i TickFallback — zapobiega zakleszczeniu triggera w Any State.

🎥 Kamera

Są dwa skrypty kamerowe — używany jest jeden z nich zależnie od konfiguracji sceny.

Główna kamera gry

CameraController

Prosta kamera izometryczna bez Cinemachine. Follow przez Lerp + zoom scrollem. Stały kąt = pełna kontrola nad wyglądem.

Cinemachine 3.x

CameraZoom

Zoom dla kamery opartej na CinemachineFollow. Skaluje FollowOffset zachowując kierunek. Cinemachine zostaje do cutscen / Timeline.

OcclusionFader — ukrywanie budynków

Skrypt na Main Camera. Gdy budynek zasłania gracza — wyłącza jego renderer i collider (gracz może klikać ziemię pod niewidoczną ścianą). Trzy filtry decydują co ukryć:

FiltrOpis
WysokośćUkrywa tylko elementy wyżej niż hideAbovePlayerY nad graczem
Odległość XZTylko w promieniu occlusionRadiusXZ od gracza
Kierunek (dot product)Tylko ściany po stronie kamery — próg occlusionDotThreshold
OcclusionFader wymagał 7+ iteracji. Nie modyfikować logiki bez dyskusji. Upgrade do płynnego fade (Shader Graph + MaterialPropertyBlock) jest planowany.

Patrol

Jeśli przypisano waypoints[] — wróg chodzi po nich w pętli. Bez waypointów — losuje nowy punkt w promieniu patrolRadius od pozycji startowej (do 8 prób na NavMesh).

Atak wroga

Wróg odpala trigger "Attack" w animatorze. Animation Event OnAttackHit() nakłada obrażenia przez playerDamageable.TakeDamage() — wróg nie wie nic o PlayerHealthComponent, tylko o interfejsie.

Stun

ApplyStun(duration) może wywołać PlayerCombat (lub dowolny system). Wróg przechodzi do stanu Stunned, NavMesh zatrzymany, po czasie wraca do Chase.

Śmierć

Stan Dead: NavMesh wyłączony, collider wyłączony, trigger animacji "Dead", publikuje OnEnemyDied { xpReward }. Gameobject niszczy się po 3 sekundach.

Efekt na statystyki

Po każdej zmianie ekwipunku RecalculateBonuses():

stats.ResetBonuses() // wyzeruj wszystkie bonus*
for each item in equipped.Values:
    stats.bonusDamage += item.bonusDamage
    stats.bonusArmor += item.bonusArmor
    // ... i wszystkie pozostałe pola bonus*

Konsumable

Przedmioty typu Consumable (np. apteczka) nie zajmują slotów ekwipunku. UseConsumable(item) wywołuje health.Heal() i/lub health.RestoreStamina() i usuwa item z plecaka.

🖥️ HUD gracza

PlayerHUD to czyste UI — tylko nasłuchuje i wyświetla. Nie woła żadnej logiki, nie modyfikuje danych. Dwa elementy graficzne:

Image — pionowy fill

Pasek HP

HealthFull nad HealthEmpty. fillAmount = currentHp / maxHp. Reaguje na OnPlayerDamaged i OnPlayerHealed.

Image — poziomy fill

Pasek Staminy

fillAmount = current / max. Reaguje na OnPlayerStaminaChanged — aktualizuje się przy każdym użyciu uniku i podczas regeneracji.

Inicjalizacja

W Start() HUD czyta aktualne wartości z PlayerStats i ustawia paski do właściwego stanu — żeby nie zaczynać od pustych pasków przy wczytaniu sceny.

🎨 Ekwipunek UI

InventoryUI to panel toggle klawiszem I. Dwa oddzielne GameObjecty: panel (zawartość) i background (przyciemnione tło) — oba toggle razem.

Układ

Plecak

20 slotów (bagSlots[])

Pokazuje przedmioty z inventory.Bag. Klik → auto-equip do domyślnego slotu lub UseConsumable.

Sylwetka

6 slotów ekwipunku

MainHand, OffHand, Helmet, Chest, Legs, Boots. Klik → Unequip (item wraca do plecaka).

Klik w plecaku — logika auto-equip

Typ itemuAkcja
Consumableinventory.UseConsumable(item) — używa natychmiast
WeaponEquip do MainHand
ShieldEquip do OffHand

InventorySlotUI

Pojedynczy slot to Image z ikoną + Unity Button. Konfigurowany przez Setup(item, callback) — callback to lambda z zewnątrz (albo equip, albo unequip). Sam slot nic nie wie o logice — tylko wyświetla ikonę i woła callback.

Odświeżanie

Panel odświeża się (Refresh()) gdy jest otwarty i dostanie event OnInventoryChanged, OnItemEquipped lub OnItemUnequipped. Gdy jest zamknięty — eventy są ignorowane (pusta klatka bez kosztów).
